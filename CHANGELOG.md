## v1.0.1.5 - 通用数据迁移预检中心

发布日期：2026-05-20

### 重要说明

- 数据迁移能力定位为“岁岁云商城通用迁移中心”，面向彩虹发卡同系站点、岁岁云商城旧版本和同系统二开站点，不做任何单站点专用绑定。
- 本版本先上线只读预检报告，不执行正式迁移，不清空目标库，不写入数据，避免误操作影响生产站。

### 修复问题

- 修复 `admin/datamove.php` 仓库版本存在乱码和线上版本不一致的风险，避免后续从 GitHub 部署时覆盖掉正常页面。
- 移除旧迁移页“一点提交就逐表清空目标库再插入”的高风险入口。

### 新增功能

- 新增迁移预检报告，支持连接源站和目标站数据库后自动识别同系商城结构。
- 新增表差异、字段差异、核心业务表覆盖、高价值字段丢失风险检查。
- 新增源库和目标库主表前缀、系统族、核心表数量和核心表行数摘要。

### 优化改进

- 将迁移判断从单纯行数比较升级为结构级检查，能发现商品价格、上游成本、锁价、订单卡密、分站余额等关键字段风险。
- 将备份表、临时表、扩展表和核心业务表分级展示，便于管理员决定哪些需要保留。

### 界面样式

- 迁移入口改为后台 shell 风格的预检页面，保留 PC 和移动端可读性。
- 使用 FontAwesome 图标和明确的风险标签，不依赖 glyphicon。

### 部署说明

- 程序缓存版本更新到 `2026052005`，后台资源版本更新到 `20260520suisuiops10`。
- 部署后需要检查 `admin/datamove.php` 的 `php -l`、HTTP 200 和后台 CSS 新缓存资源。

### 后续计划

- 下一步增加迁移方案生成：自动给出补字段 SQL、字段映射建议和可忽略备份表清单。
- 正式迁移功能需在预检报告无高风险后再开放，并加入备份、分批写入、断点续迁和迁移后业务校验。
