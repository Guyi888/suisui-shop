# 更新日志

## v1.0.2.33 - 重复购买拦截回归修复
发布日期：2026-05-26

### 重要说明

- 本版本面向岁岁云商城独立运营版本，修复上一版本在下单入口新增重复购买拦截后，部分允许复购商品被误判为重复提交的问题。
- 运营站点通过同系统 API 对接上游时，不再由 API 层按“同商品 + 同账号”强制拦截正常复购。
- 本次更新不改变数据库结构，不调整页面视觉样式。

### 修复问题

- 修复 API 对接下单时，允许重复购买的商品可能返回“今天已添加过，请勿重复提交”的问题。
- 修复前台普通下单中，历史商品 `repeat` 字段为 0 时可能误拦截正常复购的问题。
- 修复批量下单和卡密兑换入口复用同类拦截逻辑，导致同账号再次购买失败的问题。
- 清理已不再使用的 API 重复购买拦截函数，避免后续入口误调用。

### 新增功能

- 本版本不新增数据库表或独立功能模块。

### 优化改进

- 恢复运营站点复购链路的兼容性，重复购买规则不再由本次新增的 API/前台入口硬拦截扩大影响。
- 保留库存、余额、黑名单、必填项、数量上下限等原有下单校验，不改变商品库存与扣款流程。

### 界面样式

- 本版本不调整页面视觉样式。

### 部署说明

- 程序缓存版本更新到 `2026052507`。
- 本版本不包含数据库结构变更。
- 升级后建议测试同账号重复购买、API 对接补单、普通前台下单、批量下单和卡密兑换入口。

### 后续计划

- 继续梳理商品重复购买规则，把真正需要防重复提交的场景与允许复购的运营商品分开设计，避免影响正常销售。
## v1.0.2.32 - 价格层级与订单分润链路修复
发布日期：2026-05-25

### 重要说明

- 本版本面向岁岁云商城独立运营版本，重点修复多级分站价格继承、密价生效和订单分润链路不一致的问题。
- 专业版分站、普及版分站、下级分站和下级的下级现在会按同一套成本、销售价和分润口径计算，减少实际成交后利润漏记或错记。
- 本次更新不改变数据库结构，不调整前台页面样式。

### 修复问题

- 修复分站用户下单后，分润校验误用站长自购成本，导致订单成交价与校验价不一致时不写入提成流水的问题。
- 修复专业版分站下面再开下级时，下级的下级价格没有完整继承上级专业站加价模板的问题。
- 修复普通分站设置密价后，后台可见密价但实际拿货价仍可能被上级价格覆盖的问题。
- 修复多级专业分站利润拆分时，直属专业站可能拿走全部上游差价，导致更上级专业站漏记差价利润的问题。

### 新增功能

- 本版本不新增数据库表或独立功能模块。

### 优化改进

- 统一分站价格口径：对外销售价、站长自购成本、下级普通价、下级专业价和密价分别按对应语义计算。
- 多级分站分润改为逐级拆分差价，订单流水中的提成金额与每层实际拿货价保持一致。
- 分站加价模板读取上级原始商品价格后再计算，避免同一模板被重复叠加。

### 界面样式

- 本版本不调整页面视觉样式。

### 部署说明

- 程序缓存版本更新到 `2026052505`。
- 本版本不包含数据库结构变更。
- 升级后建议测试专业版分站、普及版分站、下级分站、下级的下级、密价、分站加价模板和余额下单后的提成流水。

### 后续计划

- 继续完善分站价格诊断视图，把成交价、站长成本、下级拿货价和上级利润拆开展示，方便运营核对。

## v1.0.2.30 - 手动同步任务触发修复
发布日期：2026-05-25

### 重要说明

- 本版本面向岁岁云商城独立运营版本，修复 HTTPS 站点手动同步任务创建后长期停留在“排队中 / 0%”的问题。
- 当服务器 Web PHP 禁用 `exec` 时，后台会退回本机 HTTP/HTTPS 请求触发同步接口；本版本修正该退回链路在 HTTPS 站点上的触发方式。
- 本次更新不改变自动同步的价格计算、商品落库规则、上游接口协议和数据库结构。

### 修复问题

- 修复手动同步退回 HTTP 触发时，先访问 `127.0.0.1:80` 遇到 HTTPS 跳转 `301` 却误判为已成功送达的问题。
- 修复 HTTPS 站点后台任务记录可能一直显示“任务已创建，等待后台接口接手执行”的问题。
- 修复同步任务已经创建但 `started_at` 为空、进度一直为 0%，导致运营人员误以为同步仍在排队的问题。

### 新增功能

- 本版本不新增数据库表或核心业务功能。

### 优化改进

- 手动同步触发优先使用 curl 按当前站点协议访问同步接口，并将当前域名解析到 `127.0.0.1`，减少外网回环、HTTPS 跳转和 Cloudflare 代理造成的误判。
- 在无 curl 环境下，Socket 兜底会按当前协议选择 80 或 443，不再盲目优先访问 80 端口。

### 界面样式

- 本版本不调整页面样式。

### 部署说明

- 程序缓存版本更新到 `2026052503`。
- 本版本不包含数据库结构变更。
- 升级后建议在“自动同步”页面点击手动同步，确认任务能从“排队中”进入“运行中”，并最终显示完成或明确失败原因。

### 后续计划

- 继续优化同步任务记录，把旧的排队任务、超时任务和分批同步进度展示得更清楚，方便运营人员判断是否需要继续执行下一批。

## v1.0.2.27 - 商品控价与首页推荐商品
发布日期：2026-05-24

### 重要说明

- 本版本面向岁岁云商城独立运营版本，新增商品控价金额，用于约束下级分站及更下级站点的最终前端售价最低限价。
- 控价金额只限制最终销售价，不改变对接成本、主站成本、密价成本、订单扣款和既有分润成本口径。
- 密价作为当前站长的拿货成本基准，站点启用加价模板时会继续在密价基础上计算下级专业价、下级普通价和销售价。

### 修复问题

- 修复分站存在密价时，部分路径可能把密价直接当作最终下级售价，导致站点默认加价模板未继续参与计算的问题。
- 修复后台管理侧销售价读取可能受登录态提前返回影响，误把专业价或成本价当作销售价的问题。
- 修复推荐商品接口在推荐表尚未创建时返回 500 的问题；现在会自动兼容空表并返回空推荐列表。

### 新增功能

- 后台商品编辑页新增“控价金额（最低限价）”输入项，主站可为商品设置下级最终售价底线。
- 首页新增折叠式推荐商品入口，展开后按自定义分类标签展示推荐商品。
- 推荐商品后台新增分类标签字段，支持为推荐商品配置不同展示分组。

### 优化改进

- 分站价格计算统一为：先确认当前站点成本基准，再按站点模板继续生成下级专业价、普通价和销售价，最后对最终销售价应用控价兜底。
- 批量调价保存时补充控价金额校验，避免分站把销售价批量调到主站限价以下。
- 推荐商品前台接口会自动确保推荐表结构存在，便于新站或重铺测试环境直接启用。

### 界面样式

- 首页推荐商品弹窗改为折叠入口，展开后显示分类标签和商品卡片，PC 与手机端均保持可扫描的卡片布局。
- 推荐商品按钮和弹窗图标统一使用 FontAwesome，不引入 glyphicon。
- 商品编辑页控价输入框补充运营提示，说明限价只约束最终售价，不影响成本和分润计算。

### 部署说明

- 程序缓存版本更新到 `2026052417`。
- 本版本新增 `pre_tools.min_price` 字段，并新增/补齐 `pre_recommend.group_name` 字段；运行时会自动检查并补齐字段。
- 升级后建议测试主站商品控价保存、直属分站价格、下级分站价格、批量调价和前台下单显示价。
- 推荐商品功能建议在后台新增至少一个推荐商品后，测试首页折叠入口、分类标签切换和商品直达购买页。

### 后续计划
